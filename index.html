3

Install the USB Driver

The Rockchip USB driver must be installed on your PC before the flashing tool can communicate with the board. This is a one-time setup step — skip it if the driver is already installed.

  1. Open the DriverAssistant_v5.1.1 folder.
  2. The Rockchip Driver Assistant v5.1.1 window will open.
  3. Click Drive Installation and wait for the process to complete. The progress bar will fill when done.
If you have a previous version installed
Click Driver Uninstallation first to remove the old driver, then click Drive Installation to install the current version. Mixing driver versions can prevent device detection.
4

Open the Flashing Tool

  1. Open the RKDevTool_Release_v3.18 folder.
  2. The RKDevTool window will open. The status bar at the bottom will display "No Devices Found" — this is normal at this stage.
Leave this window open
Keep RKDevTool running throughout the rest of the procedure. You will watch it detect the board in the next step.
5

Put the Board into Recovery (Flash) Mode

This is the most critical step. The sequence of power-off, button-hold, and power-on must be performed in the correct order.

  1. Disconnect power from the Android board completely.
  2. Connect the Type-C cable from the board's TYPE-C port to your PC.
  3. Press and hold the RECOVERY button — keep it held down without releasing.
  4. While still holding RECOVERY, reconnect power to the board.
  5. Continue holding for approximately 3–5 seconds, then release the button.
  6. Watch the RKDevTool status bar — it should change to "Found One LOADER Device".
Device not detected? Try rotating the cable

The Type-C connector is physically reversible but Rockchip boards can be directionally sensitive. If the device is not detected:

  1. Disconnect power and the Type-C cable.
  2. Rotate the Type-C connector 180° and plug it back in.
  3. Repeat the recovery sequence from Step 1.

If still not detected after rotating, uninstall and reinstall the driver (Section 3), then try again.

6

Load and Flash the Firmware

With "Found One LOADER Device" confirmed, follow these steps in RKDevTool. The diagram below shows the numbered click order.

  1. ① Click "Upgrade Firmware" tab at the top of the RKDevTool window.
  2. ② Click the "Firmware" button. A file browser will open.
  3. ③ Select the firmware file — navigate to and open allupgrade_3588_64G_8G. The file path will appear in the Firmware field, confirming it is loaded.
  4. ④ Click "Upgrade" to begin flashing.

The log panel on the right will show progress. A successful flash produces this sequence:

Do not interrupt the flashing process
Do not disconnect the USB cable or remove power while the firmware is downloading. Interrupting the flash can leave the board in an unbootable state.
7

Verify Boot and Perform Factory Reset

After the flash completes, the board reboots automatically into Android. A factory reset is required to clear any leftover data from the previous firmware.

  1. Wait for the board to finish booting. The Android home screen should appear.
  2. On the Android device, navigate to Settings → System → Factory Reset.
  3. Review the warning dialog and tap Confirm.
  4. Wait for the device to reset and reboot. The process is complete.
Flashing complete
The board is now running the new firmware with a clean factory state. It is ready for configuration.
Why factory reset is required
A firmware flash replaces the system partition but does not automatically wipe user data. Leftover settings and cached data from the previous firmware version can cause unexpected behaviour, so a factory reset is always required after flashing.

Troubleshooting

Problem Solution
"No Devices Found" after entering recovery mode Rotate the Type-C cable connector 180° and retry the recovery sequence (Section 5). This is the most common cause of detection failure.
Device still not detected after rotating cable Uninstall and reinstall the Rockchip driver via DriverInstall (Section 3), then retry the full recovery sequence.
RKDevTool cannot open the firmware file Confirm the file is the correct .img for this board: allupgrade_3588_64G_8G. Ensure it is not compressed (.zip) — extract it first if needed.
Flash fails or stops partway through Check that the USB cable is firmly connected at both ends. Close RKDevTool, repeat the recovery entry sequence (Section 5), then start the upgrade again (Section 6).
Board does not boot after flashing Repeat the entire procedure from Section 5. If the board remains unresponsive, confirm the firmware file matches the board model (64G storage, 8G RAM variant).
Android boots but behaves unexpectedly Ensure the factory reset (Section 7) was completed. Old cached data from a previous firmware version is the most common cause of post-flash issues.
